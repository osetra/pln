# CLI справка

> Последнее обновление: 2026-05-06

## Команды

| Команда    | Описание                          |
|------------|-----------------------------------|
| `list`     | Показать задачи (по умолчанию)    |
| `add`      | Создать задачу                    |
| `edit`     | Редактировать задачу (нужен `-u`) |
| `delete`   | Удалить задачу (нужен `-u`)       |
| `config`   | Открыть конфиг в редакторе        |
| `tui`      | Запустить интерактивный TUI       |

## Флаги задачи (TaskParams)

| Флаг | Короткий | Тип    | Описание                   |
|------|----------|--------|----------------------------|
| `--summary`    | `-t` | string | Название задачи       |
| `--categories` | `-c` | array  | Категории (можно несколько раз) |
| `--status`     | `-s` | string | Статус (NEEDS-ACTION, COMPLETED, CANCELLED) |
| `--description`| `-d` | string | Описание              |
| `--priority`   | `-pr`| number | Приоритет              |
| `--uid`        | `-u` | string | UID задачи             |
| `--parent`     | `-P` | string | UID родительской задачи |
| `--predecessors` | `--pre`, `--after` | array | UID задач-предшественников (DEPENDS-ON). Можно повторять флаг |
| `--due`        | `-e` | date   | Дедлайн                |
| `--start`      |      | date   | Дата начала            |
| `--x`          |      | bool   | Пометить COMPLETED     |
| `--o`          |      | bool   | Только открытые (применяет `config.defaultFilter`) |
| `--cancel`     |      | bool   | Пометить CANCELLED     |

## Флаги фильтрации (FilterParams)

| Флаг | Короткий | Тип    | Описание                   |
|------|----------|--------|----------------------------|
| `--all`           | `-a`   | bool   | Все задачи (без дефолтного фильтра) |
| `--add`           |        | array  | Расширить выборку (OR внутри поля)   |
| `--and`           |        | array  | Сузить выборку (AND между условиями) |
| `--only`          |        | array  | Оставить только совпавшие            |
| `--del`           |        | array  | Исключить совпавшие                  |
| `--childrensLevel`| `-l`   | number | Глубина дочерних (0, 1, 2...)        |
| `--parentsLevel`  | `-p`   | number | Глубина родителей (0, 1, 2...)       |
| `--levelAll`      | `-L`   | bool   | Все дочерние уровни                  |
| `--parentsAll`    | `-up`  | bool   | Все родительские уровни              |

## Флаги управления (ControlParams)

| Флаг | Короткий | Тип    | Описание                        |
|------|----------|--------|---------------------------------|
| `--count`        | `-n` | number | Кол-во элементов (для delete)    |
| `--hide`         | `-h` | bool   | Не печатать задачи               |
| `--raw`          | `-r` | bool   | Вывод в JSON                     |
| `--verbose`      | `-v` | bool   | Подробный вывод                  |
| `--clientFilter` | `-f` | bool   | Фильтрация на клиенте           |
| `--backendFilter`| `-b` | bool   | Фильтрация на сервере           |
| `--showDescription`| `-sd` | bool | Печатать описание под каждой задачей |
| `--fullUid`      | `-fu`| bool   | Печатать полные uid (без shortUid) — для edit при коллизиях |
| `--ink`          |      | bool   | Запустить экспериментальный TUI на ink (`npm i ink react`) |

## Типы фильтров

Фильтры принимают формат `поле:значение`.

- **add** — расширяет выборку (OR). Внутри одного поля: любое совпадение. Между полями: AND.
- **only** — сужает выборку. Оставляет только совпавшие.
- **del** — исключает совпавшие из результата.
- **and** — строгое пересечение (AND).

Порядок применения: add → only → del → иерархия (parents/children).

## Дефолтный фильтр

Когда юзер запускает `pln` без флагов (или с `-o`), применяется `config.defaultFilter` — массив условий формата `Condition` из `src/dto/filter.js`.

```jsonc
// config.json
"defaultFilter": [
  { "field": "status",     "value": "NEEDS-ACTION", "combineType": "only" },
  { "field": "categories", "value": "trash",         "combineType": "del" }
]
```

`combineType`: `add` — OR, `only` — AND, `del` — исключить.

## Примеры

```bash
# Список задач (по умолчанию — config.defaultFilter, обычно status:NEEDS-ACTION)
pln

# Поиск по названию
pln -t купить

# Все задачи
pln -a

# Завершённые задачи
pln -x

# Добавить задачу
pln add -t "Новая задача" -c work -pr 1

# Добавить задачу с многострочным описанием
pln add -t "Тест" -d "первая строка
вторая строка
третья строка"

# Показать список с описаниями под задачами
pln -t тест --sd

# Добавить подзадачу
pln add -t "Подзадача" -P uid-родителя

# Редактировать задачу
pln edit -u uid-задачи -t "Новое название"

# Завершить задачу
pln edit -u uid-задачи --x

# Удалить задачу
pln delete -u uid-задачи

# Фильтр: только проекты, исключая отменённые
pln --add categories:project --del categories:cancel

# Показать с дочерними на 2 уровня
pln -t задача -l 2

# Показать всю иерархию вверх
pln -t задача -up

# TUI с фильтром
pln -t work tui
```
